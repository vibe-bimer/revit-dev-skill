## Versioning / archive policy

- Keep the latest recommended version in active use (currently `v5-light`).
- Older topology variants (`v2`, `v3`, `v4`, legacy ppt/heatmap/roadmap assets) are retained for history and comparison.
- When introducing a new baseline version, update this README first, then update upstream references in skill docs.
